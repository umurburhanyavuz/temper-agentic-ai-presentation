Pillar 3

Context Engineering

The quality of AI output depends entirely on the context you give it.

Pillar 3 — Context

What Is Context?

Context is everything AI knows when it's working for you.

You're already doing it
Every time you paste text into a chat, add instructions to a prompt, or explain background to AI — you're managing context manually.
The goal: automate it
Instead of copy-pasting every time, set up AI so it already has access to the right information — automatically, persistently, efficiently.
It has a limit
Context windows are finite — you can't dump everything in. Quality beats quantity.
You can monitor it
Most tools show how much context is being used. Keep it relevant — not maxed out.
Better context = better results
Without enough context, AI hallucinates. With the right context, it's remarkably accurate.
Pillar 3 — Context

The 4 Layers of Context

Context = everything AI knows when it works for you. You're already managing it manually every time you paste text into a chat. The goal is to make it automatic, persistent, and richer.

Data Safety

The Hidden Dangers

The same "agency" that makes these tools powerful also makes them dangerous.
Because agents can browse, read, and write — they can accidentally become super-leakers.

Proprietary Data "Sponge"
If you upload a sensitive strategy doc to an agent to "summarise" it, that data may live in the agent's memory or be used to train a third-party model.
PII Cascading
Agents chain tasks. An agent might pull a worker's ID to verify a shift, then leak that ID into a log file or an unencrypted Slack notification.
Least Privilege
Never give an agent more access than the task requires
Human-in-the-Loop
High-stakes actions (payments, data exports) must ask a human first
Audit Trails
See not just the answer, but every step the agent took to get there
"At Temper, we give people freedom. But if our AI agents are 'too free' with our data, we lose the trust that makes our marketplace work. Agentic AI isn't just a new tool — it's a new team member that needs a background check and clear boundaries."
Data Safety

AI Data Safety Checklist

5 pillars to audit any agentic AI project before it goes live.

"Does the agent actually need to see this specific piece of data to finish the task?"

PII Redaction Layer
Sanitise all PII before it reaches the AI — replace "John Doe" with "[User_A]"
Principle of Least Privilege
Don't give agents admin keys if they only need read-only access
Human-in-the-Loop (HITL)
Financial transactions and data deletion always require manual approval
Zero-Retention & Private Context
Ensure every tool has "do not use for AI training" enabled
Step-by-Step Audit Trails
Log every tool call: "I accessed Table X, filtered by Y, output Z"
