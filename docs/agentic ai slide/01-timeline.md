# Workshop Timeline — Minute by Minute

Total duration: ~45 minutes

---

## Section 1: Opening — The Problem [0:00 - 5:00]

| Time | What | Notes |
|------|------|-------|
| 0:00 | Title slide | |
| 0:30 | "How many tools do you use every day?" — Slack, Jira, Notion, Gmail, Mixpanel, Miro... | Get heads nodding |
| 1:30 | "Now, how much time do you spend moving information BETWEEN these tools?" | The pain point |
| 2:30 | Show simple example: copy meeting notes → create Jira tickets → update Notion → message Slack | This is manual glue work |
| 3:30 | "What if AI could do this glue work — but you still approve every important step?" | Introduce the thesis |
| 4:30 | Overview of the 3 pillars: Connections, Human in the Loop, Context Engineering | Set the agenda |

## Section 2: Pillar 1 — The Data Layer [5:00 - 17:00]

| Time | What | Notes |
|------|------|-------|

| 8:30 | Show what's connected today: Notion MCP, Atlassian (Jira/Confluence) MCP, Google Workspace | Logos + connection diagram |
| 10:00 | Quick demo: Gemini reading from Google Workspace (Docs/Sheets/Gmail) | Live or screenshot |
| 11:30 | Show concept: Claude Code connected to Notion + Jira simultaneously via MCP | Screenshot or short demo |
| 13:00 | Mention Cursor — same concept, but inside your code editor | Brief, don't deep-dive |
| 14:00 | Google Workspace Studio — no-code agents that connect to Jira, Salesforce, etc. | Brief overview |
| 15:00 | Gemini CLI — for those who want terminal-based agentic AI | Brief mention |
| 16:00 | Key takeaway: "The tools exist today. The question is how we use them." | Transition |

| Time | What | Notes |

| 19:30 | Example 1: AI drafts a status report from Jira data → human reviews and sends | Walk through the flow |

## Section 4: Pillar 3 — Context Engineering [27:00 - 37:00]

| Time | What | Notes |
|------|------|-------|
| 27:00 | "Why does AI sometimes give great answers and sometimes terrible ones?" | Hook |
| 27:30 | Answer: "Context. The quality of what you get out depends entirely on what you put in." | Core concept |
| 28:30 | Prompt engineering vs. context engineering — the evolution | Simple comparison |
| 29:30 | The 4 layers of context: Instructions, Knowledge, Memory, Tools | Visual diagram |
| 31:00 | Practical example: a Gemini Gem with instructions only vs. one with full context | Show the difference |

| 34:00 | Growing context over time — start small, add as you learn | Iterative approach |
| 35:00 | Sustainability: why structured context matters for team-wide adoption | If it's in your head, it dies with your access |
| 36:00 | Key takeaway: "Context is an investment. Build it once, benefit forever." | Transition |
