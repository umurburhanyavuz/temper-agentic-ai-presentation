## Slide 14 — What Is MCP?

- MCP = Model Context Protocol. Think USB-C for AI.
- Before USB-C, every device had its own cable. MCP is the same idea: one protocol, every tool.
- Open standard — not locked to one vendor.
- 100+ tools already supported and growing.
- Agents don't just pull data — they create, update, and act.

## Slide 32 — The Hidden Dangers
*[Reveals: danger cards, then mitigation cards]*

- The same "agency" that makes these tools powerful also makes them dangerous.
- **Proprietary Data "Sponge"**: Upload a sensitive doc → data may live in the agent's memory or be used for training.
- **PII Cascading**: Agents chain tasks. A worker's ID pulled for one task could leak into a log or Slack notification.
- Three mitigations: **Least Privilege** (minimum access), **Human-in-the-Loop** (high-stakes actions need approval), **Audit Trails** (see every step).

## Slide 34 — AI Data Safety Checklist

- Five pillars to audit any agentic AI project:
- **PII Redaction Layer**: Sanitise PII before it reaches AI.
- **Principle of Least Privilege**: Don't give admin keys if you only need read-only.
- **Human-in-the-Loop**: Financial transactions and data deletion always need manual approval.
- **Zero-Retention & Private Context**: Ensure "do not use for AI training" is enabled.
- **Step-by-Step Audit Trails**: Log every tool call.
- The guiding question: "Does the agent actually need to see this specific piece of data to finish the task?"
